O que é Desenvolvimento Web Full Stack? Deixe-me começar esclarecendo
alguns termos, então começaremos com um entendimento
comum sobre essa área. Primeiro, nós ouvimos pessoas falando
sobre Front end e Back end, ou às vezes o Lado do Cliente e
o Lado do Servidor. Agora, quando falamos sobre Front end
estamos primeiramente falando sobre apresentar dados para os usuários. Isso é onde os usuários acessam a rede
através do navegador. E estaríamos usando tecnologias como
HTML, CSS e JavaScript para atingir os usuários finais. O Back end é mais relacionado a lógica do
 negócio e os aspectos de dados da nossa aplicação web, então isso poderia ser
implementado em um entre muitas linguagens. Nós normalmente escutamos pessoas
falarem sobre a Arquitetura Três Camadas para desenvolvimento web e implementação. Agora nesta abordagem, que é um padrão
comum de arquitetura de software cliente-servidor, o desenvolvimento completo 
é dividido em três camadas. A camada de Apresentação, que é relacionada 
às questões de interface  de usuário e como apresentar os dados para o usuário final. Então nós temos a camada de Lógica 
de Negócios que é mais relacionada com a validação de dados e processamento
de dados a fim de gerar informação para os usuários. E temos a camada de acesso a dados, que lida com persistência de dados e armazenamento de dados nos bastidores, 
por exemplo, em um banco de dados Explorando isso mais pra frente,
vamos ver o que é implementado em cada uma dessas 
três camadas em mais detalhes. Agora, a camada de lógica de negócios é 
tipicamente implementada, tradicionalmente, em um servidor, onde nós usamos
linguagens como Ruby, Python, C++, Java, ASP.net, para implementar um servidor. Este servidor por sua vez, está falando com um 
sistema Back end de gerenciamento de banco de dados para transferir dados e 
que por sua vez, estarão gerando a visualização Front end de 
dados para o usuário pela renderização, no lado do Servidor,
do HTML, CSS e JavaScript. Essa tem sido a abordagem tradicional que vem sendo usada com sucesso
no desenvolvimento. Nesta abordagem, nós precisamos de
especialistas em cada uma dessas três camadas. Você tem um especialista Fron end, que teria 
um bom conhecimento em HTML, CSS, e JavaScript. E então você tem os especialistas Back end, que tem bons conhecimento em
implementações do lado do Servidor, quer dizer, por exemplo, em uma das linguagens usadas
para implementação do lado do Servidor. E então você teria um especialista de dados
que seria muitas vezes relacionado com os bancos de dados e o elemento de persistência
 de dados do nosso desenvolvimento web. Existe uma tendência crescente relativa ao 
Desenvolvimento Web Full Stack onde uma única linguagem é usada para 
implementação através da pilha toda. Você poderia ter o front implementado 
como uma aplicação de página única, usando um dos frameworks 
JavaScript como AngularJS. Então o lado do Servidor poderia ser implementado 
usando tecnologias como NodeJS e módulos  NodeJS, que novamente,
dependem de JavaScript. E o Acesso de Dados ou a parte do 
armazenamento de dados implementados usando tecnologias como MongoDB, que trabalha
baseado no armazenamento de dados JSON, e fornecendo informação no
 formato de dados JSON. Agora, JSON se tornou o formato padrão para intercâmbio de dados entre as três camadas. Lado do Servidor está cada vez mais
entregando uma API REST, assim você pode atingir múltiplas
plataformas através da API REST. Essencialmente, o servidor está
fornecendo dados na forma de JSON, que podem ser renderizados tanto no
padrão do navegador web, ou no dispositivo móvel usando uma
entre múltiplas formas, ambientes de desenvolvimento 
de aplicativos móveis híbridos. Então, nesta especialização em particular,
estamos atingindo a pilha inteira. Veremos sobre HTML, CSS e JavaScript e vamos analisar o desenvolvimento de
interface web usando Bootstrap. Então, veremos os frameworks Front end
JavaScript como AngularJS. Veremos sobre NodeJS e
módulos NodeJS, e também revisaremos Back end como
um serviço, e também revisaremos suporte de dados através de
MongoDB no lado do Servidor.
