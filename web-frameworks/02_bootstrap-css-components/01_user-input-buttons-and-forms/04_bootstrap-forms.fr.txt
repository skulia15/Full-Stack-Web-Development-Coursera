Parlons maintenant de l'aide de Bootstrap pour les formulaires. En particulier, les différentes classes que Boostrap met à notre
disposition pour la mise en forme des formulaires. Nous avons déjà vu en détail les formulaires
dans le cours précédent. Donc dans ce cour, je vais
principalement parler des classes que Bootstrap met à notre disposition
pour la mise en forme des formulaires. Dans Bootstrap, les formulaires sont créés
en utilisant les balises des formulaires. Donc tous les éléments de mise en forme
sont inclus dans les balises du formulaire. Si aucune classe n'est spécifiée,
alors c'est la présentation standard du formulaire qui est utilisée. Nous pouvons préciser que la classe du
formulaire est "form-horizontal". En faisant cela, nous pouvons
obtenir que les "labels" seront affichés à gauche des champs
pour la saisie des données. Donc comme vous pouvez le voir
dans cet exemple, en spécifiant ensemble un label et un
champ de saisie nous pouvons les regrouper en
utilisant la classe "form-group". Donc ici, je spécifie le label
comme étant le prénom (firstname) et qui est affiché du coté gauche 
dans le formulaire. Le champ de saisie est affiché sur la même ligne, du coté droit. Maintenant, la classe "form-group" de Bootstrap prend en charge la classe "row". À l'intérieur d'une balise utilisant "form-group"
vous pouvez utiliser les classes "col-*" afin de réaliser la mise en place
des éléments. Ici, vous pouvez voir que j'utilise
les classes de colonnes, aussi bien pour le label que pour la zone de saisie. Nous pouvons définir des formulaires
comme étant "inline". Les formulaires "inline" sont
affichés sur une unique ligne. Ceci peut être très utile,
par exemple si vous souhaitez ajouter dans votre barre de
navigation, la possibilité pour l'utilisateur de saisir son nom et son 
mot de passe pour se connecter au site. Donc les formulaires en ligne
utilisent la classe "form-inline" et à l'intérieur vous pouvez
indiquer des groupes. En particulier dans cet exemple,
vous voyez que pour le label j'indique l'utilisation de la classe "sr-only"
(Screen Readers only). ce qui veut dire que ces labels ne sont
valides que pour la lecture sur écran. Ainsi, pour un affichage normal,
ces labels seront cachés. À la place, nous allons utiliser l'attribut "placeholder"
directement dans l'élément du formulaire pour indiquer ce que l'utilisateur est
supposé écrire dans ce champ. Une autre classe prise en charge par Bootstrap
est appelée comme un "input-group". Un "input-group" vous permet de
regrouper un ensemble d'éléments. Maintenant, pour tout champ de saisie que
vous avez dans votre formulaire, vous pouvez utiliser "input-group-addon", 
ce qui vous permet ainsi d'ajouter des éléments à gauche et à droite de votre groupe. Ainsi dans cet exemple, vous voyez
que j'ai ajouté des parenthèses gauche et droite autour du champ où je permets à 
l'utilisateur de saisir son code. Par exemple pour un numéro de téléphone. Ici j'utilise deux "input-group-addon"
pour joindre à ce groupe la parenthèse gauche et la parenthèse droite. J'utilise les "group-addon" dans des balises div pour les regrouper avec le champ de saisie. Nous allons maintenant passer à l'exercice suivant, où nous allons continuer à 
travailler sur notre site. Nous allons ajouter des boutons sur le site web, nous allons également utiliser divers éléments
afin de créer un formulaire de commentaires des utilisateurs pour notre restaurant.
